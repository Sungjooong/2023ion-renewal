INEXTA COMMERCE 개요

INEXTA COMMERCE는 B2C, B2B, B2E등 공급사, 판매사, 플랫폼, 사용자 간 다양한 서비스 기능을 제공하는 오픈소스/MSA 기반의 e-Commerce 통합 개발 프레임워크 입니다.
Headless Commerce를 지향하는 INEXTA COMMERCE는 다양한 Frontend 기술들을 API를 통해, 환경이나 디바이스 제약없이 간편한 연동이 가능합니다.
또한, MSA(Microservice Architecture)를 지향하고, DevOps 운영환경을 지원해 안정적이고 한결같은(Seamless)서비스 품질을 보장합니다.

INEXTA COMMERCE 도입 기대효과

e-Commerce 전 영역에 대응하는 MSA 이머커스 플랫폼

중견급 이상 범용 이커머스/오픈마켓 비즈니스 플로우 탑재로 완결성 높은 서비스 제공

- 초기 구축비용 절감 / 빠른 구축 기간 / 고객 맞춤 컨설팅
- 스프링부트 기반의 이커머스 프레임웍 제공 / 가이드에 따른 프레임웍 변경/수정 가능
- 다양한 고객의 비즈니스 로직/프로세스에 맞는 유연한 수정가능

업종과 규모에 관계없이 최적화된 서비스 설계 및 구축

- SaaS 플랫폼 도입으로 초기 구축 비용 절감
- 다양한 형태의 신속한 인프라 구성
- 오토스케일링(Auto Scaling)을 통한 무 중단 시스템 확장

강력한 Java 스프링부트 기반 이커머스 플랫폼

- 대용량 처리가 가능한 고성능 플랫폼
- 강력한 Java 설계로 쇼핑몰의 안전성과 보안성 강화
- Java Spring/MyBatis Framework 적용

라이브 쇼핑부터 상품 추천까지 확장 기능 제공

- WebRTC(Web Real-Time Communication)를 통한
- 라이브 스트리밍과 채팅 및 상품 구매 가능
- 상품 추천, 관련 상품 추천 등 개인화 영역 특화 구성
- SNS 회원가입, 간편 로그인 기능 적용

운영과 마케팅을 위한/ 최적의 자체 제공 솔루션

- 상품과 연계된 기획전, 이벤트 등 다양한 형태의 컨텐츠 생산 가능
- 품질 손상 없는 상품 이미지 자동 변환 (리사이징 & 크롭)
- Google Analytics, Firebase 등 통계 시스템 연동 제공

서비스 모듈 간 독립실행 및 조합 실행

INEXTA COMMERCE는 공통(Core, Util, Auth) 모듈을 기반으로 각 Service 모듈을 독립적으로 실행될 수 있으며, Service 모듈을 Grouping하여 실행할 수도 있습니다.

INEXTA COMMERCE는 Service 모듈을 조합하여 다양한 MSA 서비스를 구성할 수 있습니다.

MSA Database 구성

INEXTA COMMERCE는 Service 모듈에 따라 DB를 분리 시 소스 수정이 필요하지 않습니다.
DB 분리 시 다른 업무에서 필요한 테이블은 Kafka 를 통해 복제하여 소스에 영향을 주지 않습니다.

Log 처리 및 관리

[Log 종류별 처리]
1. 중요(회원/결제/..) Log: Kafka
- 회원/결제 등 변경이력을 DB 저장 필요한 Log용(Kafka Appender)
- Kafka에 전송된 Log를 읽어 DB 저장(Kafka Log To Database: Log 처리 전용 Application)

2. 개발자용 Log: ELK(Logstash/ElasticSearch/Kibana)
- Application Log를 Logstash로 전송
